# 蓝筱玉学习中心第一阶段信息架构设计

## 目标

在不改变现有深色铜金视觉和课程、工具、认证、权限下载功能的前提下，把学习中心从资料浏览页升级为“学习—练习—提交—反馈—通关—成长”的可扩展骨架。

## 第一阶段范围

- 登录后的默认工作区改为“今日学习”。
- 一级导航改为今日学习、我的课程、作业与通关、实战工具、成长档案、资料库。
- 电脑端保留横向铜金导航；手机端新增底部五项导航。
- 将课程卡片按课程权限显示，并提供进度、最近内容、未完成作业和继续学习入口。
- 新增作业与通关页面的状态、要求、提交入口、反馈区域骨架。
- 抽出公共类型、课程数据、工具数据和导航配置；保留现有工具计算逻辑。
- 输出 Supabase 迁移草案，不删除现有 courses、user_course_access 或 auth.users。

## 暂不纳入

- 不在本阶段实现真实图片上传、教师批改后台和通知系统。
- 不迁移或删除现有课程资料。
- 不将私有 PDF 移入 public。

## 信息架构

```text
学习中心
├─ 今日学习 / dashboard
├─ 我的课程 / courses
│  └─ 课程详情 / courses/[courseId]
├─ 作业与通关 / assignments
├─ 实战工具 / tools
├─ 成长档案 / growth
├─ 资料库 / resources
└─ 个人账户 / profile
```

## 数据边界

- `types/learning.ts`：导航、课程、作业、进度和反馈类型。
- `data/courses.ts`：现有课程与课程内容，不包含页面渲染逻辑。
- `data/tools.ts`：现有工具和模板索引。
- `components/layout/`：桌面导航、手机底部导航和页面壳。
- `components/learning/`：今日学习、课程卡片、作业卡片、进度展示。
- `app/lib/`：Supabase 客户端和权限读取保持兼容。

## 成功标准

1. 登录学员默认看到今日学习。
2. 未授权课程不出现在我的课程列表的可学习区。
3. 课程主题、工具、作业、通关标准仍可分别打开。
4. 原有受保护下载入口仍经过权限检查。
5. 手机端底部导航可切换五个核心区。
6. 现有视觉变量和背景资源不被替换。
